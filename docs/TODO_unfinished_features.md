# FocusGS 未完成功能清单

更新时间：2026-05-05

## 1. 训练链路与后端托管

- 真实训练进程的脱离会话托管还没完成。
目前训练仍容易受到前端开发服务重启、终端会话中断的影响，导致“训练状态已失联”。

- 训练状态恢复机制还不完整。
刷新页面后，前端能恢复部分任务信息，但如果训练进程本身不再受控，仍无法稳定续接真实运行状态。

- 图片目录模式与单个视频模式的真实训练入口还没接通。
当前最小可运行版主要打通的是 `COLMAP` 模式，图片目录和视频模式仍以工作流展示和前端逻辑为主。

- 视频输入的真实抽帧流程还没接入后端执行。
前端已有模式与时间线，但还没有把 `ffmpeg/抽帧 -> COLMAP -> MEGS² 训练` 串成真实可运行任务。

- 图片目录输入后的真实 COLMAP 流程还没接入后端执行。
目前缺少“上传图片目录后自动跑 COLMAP，再进入训练”的完整后端任务链。

- 训练完成后的真实结果导出链路还没打通。
当前工作台里的“结果导出演示分支”是前端演示，不是真实读取训练输出并完成保存。

## 2. 历史记录与断点续训

- 历史记录的业务仍需继续打磨。
当前已经有任务列表、状态、展开详情、断点续训入口，但整体还没做成完整的实验管理页。

- 断点续训的真实稳定性还需要继续验证。
当前续训命令会带 `--start_checkpoint`，前端显示也已改成绝对迭代进度，但还需要更多实际任务验证。

- checkpoint 保存策略还不够理想。
目前更适合再补成“按固定间隔保留多个 checkpoint”，让续训来源更稳定、历史实验也更完整。

- 历史记录与真实输出文件之间的联动还不完整。
例如：从历史任务直接加载最终结果、查看导出目录、对比多个任务结果，这些还没做完。

## 3. 监控与实验分析

- WandB 接入还没有完成验证。
目前还没有确认 `MEGS²` 训练过程是否已经稳定接入 WandB，也没有前端配套展示。

- 训练过程中的更多指标看板还没做真实化。
当前主要展示 `loss / speed / eta`，但更多训练分析项还没有形成完整监控面板。

## 4. 首页工作流与展示内容

- 首页最后一个工作流屏虽然已经接入四段视频，但文案还没全部最终定稿。
当前只明确修改了第一张和第四张，第二、第三张仍可继续优化。

- 首页四卡片的视频与文案还需要整体联调一次。
包括切换节奏、字幕长度、移动端裁切、与页面留白节奏的一致性。

- 首页“结果导出”与“历史记录”卡片后续还可以补更强的视觉引导。
例如更明确的导出结果样式、断点续训强调点等。

## 5. 结果导出与轻量化保存

- 真实的结果保存界面还没和训练完成事件打通。
当前 `x` 触发的是隐藏演示分支，不是真实训练完成后自动出现。

- `Output Size / Compressed Size` 目前仍是演示值。
后续需要改成真实读取导出目录大小，才能形成可信的展示。

- 轻量化资产保存的真实后处理脚本还没接到工作台。
也就是训练完成后如何生成网页友好的轻量资产，这部分还没有完整落地成自动流程。

## 6. Viewer 与结果展示

- 训练期间的实时可视化预览目前没有继续做。
当前策略已经改成“训练时只看日志，不加载中间结果”，如果后续需要恢复实时预览，还要继续设计。

- 训练完成后自动切换最终结果到 Viewer 的链路还需要继续核对。
尤其是历史任务、续训任务、导出结果与 Viewer 加载之间的一致性。

## 7. 工程与体验收尾

- 当前仓库里还有一些未整理的运行产物与临时资源。
例如 `.runtime/`、`figures copy/`、额外 gif 等，后续需要决定哪些正式纳入仓库、哪些忽略。

- 本地开发环境对视频处理工具的依赖还没系统化。
这次通过额外安装工具完成了裁剪，后续可以考虑把这类依赖写进说明文档。

- 还缺一份更完整的“当前系统架构与已实现功能说明”文档。
方便后续继续开发、答辩展示或交接实现。

## 8. 建议优先级

建议下一阶段优先顺序：

1. 训练进程脱离前端会话托管，解决状态失联问题。
2. 完整打通图片目录 / 视频输入的真实后端流程。
3. 把真实训练完成与结果导出界面打通。
4. 继续完善历史记录、断点续训与最终结果加载闭环。
5. 最后再补 WandB、更多监控与首页展示细节。
